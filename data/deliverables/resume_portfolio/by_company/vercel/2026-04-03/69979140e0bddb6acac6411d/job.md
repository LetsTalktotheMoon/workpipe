# Software Engineer, Dashboard

**Company:** Vercel
**Location:** San Francisco, CA
**Seniority:** Mid, Senior Level
**Work Model:** Onsite
**Employment Type:** Full-time

## About the job
Vercel is a company that provides tools and cloud infrastructure for developers to build a faster, more personalized web. They are seeking a Full Stack Engineer to build and own the Vercel Dashboard, which serves as the control plane for every Vercel user, focusing on performance, personalization, and integration with various product areas.

## Core Skills
- React
- TypeScript
- Next.js
- API development
- Backend services

## Minimum qualifications
- 4+ years of experience as a software engineer building user-facing products
- Strong across the stack: modern frontend (React
- TypeScript
- Next.js) and backend (APIs, databases, services)
- Built products for developers or technical users
- Understand the expectations of this audience and hold yourself to the same bar
- Think in systems
- Naturally build reusable abstractions, composable components, and clean APIs rather than one-off solutions
- Comfortable with ambiguity
- Can take a rough problem statement and turn it into shipped code
- Use AI tools in your own development workflow and have opinions on how they should work
- Ship fast without sacrificing quality

## Preferred qualifications
- Experience working on platform or horizontal teams where you built shared infrastructure that other teams depended on
- Contributions to open source projects, especially in the React or Next.js ecosystem

## Responsibilities
- Build and own core Dashboard infrastructure: the shell, navigation, layout system, and shared components that every product area builds within
- Architect and implement the backend services and APIs that power the Dashboard, including personalization, notifications, and agent-powered experiences
- Drive performance as a core feature. Profile, measure, and optimize aggressively. Every interaction should feel instant
- Build agent-facing surfaces: the UI and API layers where agents surface errors, suggest fixes, update config, and keep users on best practices
- Work closely with product area teams (Billing
- CI/CD
- Observability, etc.) to ensure their pages integrate cleanly into the Dashboard shell and follow shared patterns
- Ship incrementally and continuously. This is a living product that improves every week
