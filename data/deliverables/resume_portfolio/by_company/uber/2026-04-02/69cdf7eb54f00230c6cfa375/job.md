# Software Engineer, Observability

**Company:** Uber
**Location:** San Francisco, CA
**Seniority:** Mid Level
**Work Model:** Onsite
**Employment Type:** Full-time

## About the job
Uber is providing a centralized observability data platform that empowers engineers with tools for monitoring and troubleshooting. The role involves designing system architecture for a centralized logging system and collaborating with various teams to drive improvements in system performance and efficiency.

## Core Skills
- Backend programming languages
- Distributed backend services
- Distributed systems
- Batch data processing
- Stream data processing

## Minimum qualifications
- BS or higher degree in Computer Science, or a related technical discipline, or equivalent experience
- Proficient in one or more backend languages, like Java
- Go
- C/C++
- C#, with the ability to pick up new ones quickly
- Strong problem solving skills, with relevant experience (3+ years) in designing and implementing distributed backend services

## Preferred qualifications
- Proven record of building and operating highly reliable distributed systems at scale
- Batch and stream data processing experience is a big plus
- Experience with OpenTelemetry
- Prometheus
- Influx and/or building and operating monitoring infrastructure
- Experience building user interfaces with TypeScript is a plus
- Under the hood experience with Apache Lucene
- ElasticSearch
- OpenSearch and other Search technologies is a plus

## Responsibilities
- Design system architecture, own key components to deliver a centralized logging system for Uber
- Join on call rotation, driving continuous improvements on system availability, scalability, performance and efficiency
- Collaborate with other infrastructure teams, production engineer team and product manager to drive adoption and best practices, and to design and implement high impact, cross-product features
