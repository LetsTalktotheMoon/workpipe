# Software Development Engineer in Test (SDET II)

**Company:** Magnite
**Location:** LA Metro Area
**Seniority:** Mid Level
**Work Model:** Hybrid
**Employment Type:** Full-time

## About the job
Magnite is the world’s largest independent sell-side advertising platform, looking for a Software Development Engineer in Test (SDET) to ensure the quality, reliability, and scalability of their systems. The role involves designing, building, and maintaining automated testing solutions while collaborating with software engineers and product managers to improve test coverage and software quality.

## Core Skills
- Test automation frameworks
- API testing
- Distributed systems testing
- Programming languages - Java
- Programming languages - Python

## Minimum qualifications
- Experience as an SDET
- QA Engineer with strong automation focus, or Software Engineer with test ownership
- Strong programming skills in one or more languages (e.g.
- Java
- Python
- JavaScript
- C#, or similar)
- Hands-on experience with test automation frameworks and tools (e.g.
- Selenium
- Playwright
- Cypress
- REST-assured, pytest, etc.)
- Experience testing APIs, services, and distributed systems
- Solid understanding of software development fundamentals, data structures, and debugging
- Familiarity with CI/CD systems and version control (e.g.
- Git)
- Ability to work across multiple systems and collaborate effectively with cross-functional teams
- Strong communication skills and a proactive approach to quality

## Preferred qualifications
- Experience with cloud platforms (AWS
- GCP
- Azure)
- Experience with performance, load, or reliability testing
- Knowledge of containerization and orchestration tools (Docker
- Kubernetes)
- Experience working in Agile/Scrum environments

## Responsibilities
- Design, develop, and maintain automated test frameworks for multiple systems, services, and APIs
- Write reliable, maintainable automated tests (functional, integration, regression, and end-to-end)
- Collaborate with software engineers to embed testing best practices throughout the development lifecycle
- Participate in requirements and design reviews to provide a quality and testability perspective
- Execute and maintain test plans for complex, distributed systems
- Investigate, troubleshoot, and clearly document defects and test failures
- Improve CI/CD pipelines by integrating automated tests and quality gates
- Continuously evaluate and improve testing tools, frameworks, and processes
